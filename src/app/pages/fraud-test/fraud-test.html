Scénario IBAN différent (historique)

Importez deux PDF facture du même fournisseur (même SIREN ou même normalisation de nom) avec des IBAN différents dans le texte / extraction. La première analyse enregistre l’IBAN côté serveur ; la seconde peut déclencher le signal IBAN_DIFFERENT_HISTORIQUE (selon les règles du backend).

Limite de POST /api/documents/inject

L’injection JSON ne stocke pas de PDF dans Redis (document:id:pdf absent). Un fraud-scan exige le PDF → réponse 400 « PDF manquant ». Pour une analyse fraude complète, utilisez un upload ou un flux qui conserve le binaire.
